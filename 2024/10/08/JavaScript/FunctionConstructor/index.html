Function Constructor

发表于 2024-10-08 分类于 JS

Function

Function对象提供了用于处理函数的方法。在JS中，每个函数实际上都是一个Function 对象

new Function()

当Functioon作为一个构造函数时，会创建一个新的Function对象，该对象只能在全局作用域中运行

会有安全问题和相对较小的性能问题

参数

  • argN (可选)
    argN意味着你可以传入N个参数(N>=0)，每个名称必须都是一个字符串，对应一个有效的js参数，这些参数可以在functionBody内访问到
  • functionBody
    一个由函数体组成的字符串

如果argN不传的话，那么会默认第一个参数就是functionBody

返回值

返回一个由匿名函数组成的字符串，类似于调用toString()

与直接写函数表达式或者函数声明创建一个函数有什么区别呢？

  1. 使用new Function()创建的Function 对象会在函数创建时完成解析，而函数声明和函数表达式会和其他的代码一起被解析
  2. 使用new Function()的效率比另外两种方式更低
  3. 在new Function创建的Function对象中，如果你想在FunctionBody内访问函数名，可以通过arguments.callee来指代函数本身(非严格模式下)，而另外两者可以直接访问函数名
