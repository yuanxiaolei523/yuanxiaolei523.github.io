调用时机

Hooks只能在组件或者自定义Hook的最顶层调用，不能出现在条件语句、循环语句或者其他嵌套函数内调用

React如何知道返回哪个state

在同一组件的每次渲染中，Hooks都依托于一个稳定的调用顺序。

在React内部，为每个组件都保存了一个数组，其中数组每一项都是一个state对, 下标为0的是state值，下标为1的是setter函数

2. useReducer

功能

在组件的顶层作用域调用useReducer来创建一个用于状态管理的reducer

const [state, dispatch] = useReducer(reducer, initialArg, init?)

参数

  • reducer: 用于更新state的函数，参数为state和action，返回值是更新后的state
  • initialArg：用于初始化state的任意值。初始值的计算逻辑取决于下一个init参数，该值会传入init函数的arguments内
  • init：可选。用于计算初始值的函数，如果存在，那么initialArg无效，如果不存在，initialArg有效

返回值

useReducer返回一个由两个值组成的数组

  1. 当前state。除此渲染时，是initialArg或者init()计算的值
  2. dispatch函数。用于更新state并触发组件的重新渲染(详细见下面)

注意：

  1. useReducer是一个Hook，只能在组件的顶层作用域或者自定义hook中调用，而不能在循环或条件语句中调用
  2. 严格模式下React会调用两次reducer和初始化函数

dispatch函数

useReducer返回的dispatch函数允许你更新state并触发组件的重新渲染。它需要传入一个action作为参数

React会调用reducer函数以更新state，reducer函数的参数为当前的state与传递的action

参数：

  • action: 用户执行的操作。可以是任意类型的值。通常来说 action 是一个对象，其中 type 属性标识类型，其它属性携带额外信息。

返回值

  • 无

注意：

  1. dispatch函数是为下一次渲染而更新state，所以调用dispatch后读取state并不能拿到最新的值
  2. 如果你提供的新值与当前的state相同(Object.is比较)，React会跳过组件与子组件的渲染
  3. React会批量更新state。state会在所有时间函数执行完毕并且已经调用过它的set函数后更新，如果在访问 DOM 等极少数情况下需要强制 React 提前更新，可以使用 flushSync。

dispatch了一个action后，如何获取最新的state

正常在使用dispatch派发一个action后，获取到的state依然是旧的，这是因为dispatch函数是为下一次渲染更新state，且state也会在所有的事件处理函数都执行完毕后才会去进行更新。

这时想要获取最新的state，可以手动调用reducer来获取结果

React使用Object.is进行对比，如果相等会跳过更新，通常出现在更新数组或者对象时，建议reducer内返回新一个新的对象或者数组

useContext

功能：

在组件的顶层调用useContext 可以让你读取和订阅组件中的 context。

语法:

const value = useContext(SomeContext)

参数

  • SomeContext：先前用 createContext 创建的 context。context 本身不包含信息，它只代表你可以提供或从组件中读取的信息类型。

返回值

useContext 为调用组件返回 context 的值。他的值来自于距离当前组件最近的上方的SomeContext.Provider 的value。如果没有这样的Provider，那么其值是使用createContext创建context时传入的defaultValue。如果 context 发生变化，React 会自动重新渲染读取 context 的组件。如果 context 发生变化，React 会自动重新渲染读取 context 的组件。

注意事项：

  • 在使用useContext() 的 组件内返回的 provider 不会影响到context值。相应的 <Context.Provider> 需要位于调用 useContext() 的组件 之上。
  • 从 provider 接收到不同的 value 开始，React 自动重新渲染使用了该特定 context 的所有子级。先前的值和新的值会使用 Object.is 来做比较。使用 memo 来跳过重新渲染并不妨碍子级接收到新的 context 值。
  • 尽量不要有相同的context，这可能会破坏context。这里使用===来比较的
  • 如果在Provider中不指定value，那么会默认变成value={undefined}
